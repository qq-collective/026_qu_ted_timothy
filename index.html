QuestQueries — #026–030

QU_TED Project英語ゼミの議論を、上流から下流まで支える5つのAI。

どれか1つから始めていい。——バンドは、いつでも待っている。

使う流れ
ゼミ前  →  隼人 #030
登壇前  →  Timothy #026
発表後  →  Elice #027
議論中  →  Donovan #028
いつでも  →  Dr. Mucho #029
#026
Timothy
発表準備・骨格設計
要旨やスライドを入力すると、TED風の発表骨格と「登壇者への4つの問い」を生成。「正しく読み上げる」から「場を動かすために話す」への転換を支援する。
登壇前 →
#027
Elice
聴衆の声の可視化
発表後、聴衆の声を3カテゴリで収集しマトリックスに整理。「あなたへの質問攻め」を「みんなの地図」に変える。誰が何を感じたかが見えるから、議論が始まる。
発表直後 →
#028
Donovan
議論の記録・アーカイブ
フリートークをZoom/Teams文字起こしごと受け取って記録。「今は弱くていい。積み重ねが資産になる」設計。後から振り返ったとき意味があったとわかる記録を残す。
フリートーク中〜後 →
#029
Dr. Mucho
概念の血肉化・用語の読み合わせ
「この用語、なんか引っかかる」を持ち込むと、日本語・英語・例え話の3カテゴリで言い換え候補を展開。チームで「これがわかった！」を探すゲーム型ツール。
いつでも →
#030
隼人
論文読解・推論スタイルの発見
論文の一節を貼り付けると、演繹・帰納・アブダクションの3つの推論方式で同時に分解。どの読み方を選ぶかで、自分の思考スタイルが見えてくる。
ゼミ前の予習 →
AIは踏み台ではなく、重力を貸してくれる存在——スイングバイ。
うまく使えば、自分が思っていたより遠くに飛べる。
どこに向かうかを決めるのは、参加者自身だ。
